# CLAUDE.md

## Project Overview

oclif CLI + Claude Code plugin that runs Claude in isolated, network-hardened Docker containers.

## Tech Stack

- TypeScript: target `es2022`, module `Node16`, strict mode
- Node.js: `>=18.0.0` required
- oclif v4 for CLI framework
- dockerode v4 for Docker API
- mocha + chai for tests, ESM loader via ts-node
- ESM throughout (`"type": "module"` in package.json)

## Commands

```bash
npm run build   # rm -rf dist && tsc -b
npm test        # mocha (runs lint as posttest)
npm run lint    # eslint
```

## Coding Patterns

### Commands
- Files in `src/commands/`, one file per command
- Extend `Command` from `@oclif/core`
- Static `flags` and `args` properties define the CLI interface
- Call `this.parse(CommandClass)` in `run()` to destructure flags
- Use `this.log()` for output, `this.error()` to exit with error

### Docker
- All Docker operations go through `SandboxDocker` class in `src/lib/docker.ts`
- Never call dockerode directly from commands — instantiate `SandboxDocker` and use its methods

### Config
- `src/lib/config.ts` — `loadConfig()` returns `{...DEFAULTS, ...partial}` (user JSON merged over defaults)
- Config file lives at `~/.claude-sandbox/config.json`
- Use `loadConfig()` at the top of `run()` in commands

### Container Discovery
- Docker label `app=claude-sandbox` (`LABEL_KEY`/`LABEL_VALUE`) is applied to all managed containers
- `listContainers()` filters by this label; don't filter by name patterns

### Container Naming
- Format: `claude-sandbox-{name}-{hex}` generated by `src/lib/container-name.ts`
- Use `generateContainerName(repo, name?)` — don't construct names manually

## Testing

- `test/lib/` — unit tests for pure functions, no Docker required
- `test/commands/` — command tests that stub Docker interactions
- `test/integration/` — require Docker running, spin up real containers
- Framework: mocha + chai, ESM loader via `ts-node/esm`
- Timeout: 60 000 ms (set in `.mocharc.json`)

## What Not To Do

- Don't run container processes as root — only `init-firewall.sh` runs via sudo
- Don't bypass or weaken firewall allowlist rules in the Docker image
- Don't hardcode GitHub tokens — use `config.githubPat` or `gh auth token` via `execFile`
- Don't mount the Docker socket (`/var/run/docker.sock`) into containers
